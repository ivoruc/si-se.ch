Willkommen

Technische Schuld (Technical Debt) ist eine Metapher für mögliche Konsequenzen von schlechter technischer Umsetzung von Software (siehe Wikipedia: Technische_Schuld). Es ist ein immer populärer werdendes Mittel, um Stakeholdern von Software-Projekten frühzeitig den zusätzlichen Aufwand zuzeigen, welcher längerfristig durch Unterlassungen in Qualitätssicherung und Software-Entwicklung entsteht.
